## 五、各目录/模块详细作用

### edu-gateway（网关）
API 统一入口，所有 `/achievement/**`、`/system/**` 等请求通过此网关路由到对应微服务。内含 XSS 过滤、验证码路由、黑白名单配置。

### edu-auth（认证中心）
处理用户登录、登出、令牌刷新、验证码生成。不直接访问业务数据库，通过 Feign 调用 edu-system 验证用户。

### edu-api（远程接口）
定义供其他模块跨服务调用的 Feign 接口。目前仅有 `edu-api-system`（用户/角色/日志远程接口）。

### edu-common（9 个公共库）
- **core**: AjaxResult、BaseController、全局异常处理、Excel 工具、文件工具
- **security**: sa-token + JWT 认证拦截、@RequiresPermissions 注解
- **redis**: RedisTemplate 封装、分布式锁
- **log**: @Log 注解 + AOP 切面记录操作日志
- **datascope**: 数据权限 SQL 拦截器（按部门/角色过滤数据）
- **datasource**: 多数据源 + @DS 注解切换
- **seata**: Seata AT 模式分布式事务
- **swagger**: Springdoc OpenAPI 自动文档
- **sensitive**: 数据脱敏注解（手机号/身份证/邮箱）

### edu-modules/edu-achievement（核心业务）
- **domain**: `EduAchievement`（成果实体，含 @Excel）、`EduAuditRecord`（审核记录）
- **mapper**: MyBatis 接口 + XML（含 LEFT JOIN sys_user、4 条聚合统计 SQL）
- **service**: 业务逻辑层（CRUD + 聚合统计 + 站内通知）
- **controller**: `EduAchievementController`（管理员/教师 CRUD）、`EduAuditRecordController`（院级/校级审核 + 统计）

### edu-ui/src/views/achievement（6 个业务页面）
- **achievement**: 管理员成果管理（CRUD 表格 + 富文本编辑器）
- **myAchievement**: 教师申报表单（带文件上传和富文本）
- **myAchievementList**: 教师成果列表（Tabs 按状态筛选 + 统计卡片）
- **collegeAudit**: 院级审核工作台（渐变头部 + 审核弹窗）
- **schoolAudit**: 校级审核工作台（支持按学院过滤）
- **auditRecord**: 审核记录查看/导出

### docker
Docker Compose 编排 10 个服务：MySQL 5.7、Redis、Nacos 配置中心、Nginx 反向代理、Gateway、Auth、System、Gen、Job、File、Monitor。

## 六、建议与待开发优化方向

### 🔴 紧急修复（影响核心功能）

**1. 补全审核弹窗「退回修改」选项**
在 `collegeAudit/index.vue` 和 `schoolAudit/index.vue` 的 `el-radio-group` 中增加第三个单选按钮 `<el-radio label="3">退回修改</el-radio>`。同时修改 `submitAudit()` 中的确认文本逻辑，处理 `auditResult === '3'` 的情况。这是 status=5 流程的前端入口，当前后端已支持但前端无法触发。

**2. 教师页支持 status=5 重新编辑**
在 `myAchievementList/index.vue` 中增加 `<el-tab-pane label="退回修改" name="5">` 标签页。为 status=5 的成果行增加"重新编辑"操作按钮，点击后打开编辑表单。教师提交后应将 status 设回 "1"（重新进入院审），需在 `EduAchievementController` 中增加 `teacherResubmit()` 方法（校验 status 必须为 5）。

**3. Docker 目录重命名**
将 `docker/ruoyi/` 物理目录重命名为 `docker/edu/`，否则 docker-compose build 时 context 路径（`./edu/gateway` 等）找不到 Dockerfile。

### 🟡 功能增强（提升用户体验）

**4. 教师统计数据隔离**
当前 `/audit/statistics` 接口返回全局数据，教师角色应只看到自己的成果统计。可在后端增加 `role` 参数或 `teacherId` 过滤，教师端仅查询 `teacher_id = 当前用户ID` 的成果。

**5. 学院名称映射**
Dashboard 柱状图的学院轴目前显示 `college_id` 数字，应在后端 SQL 中 LEFT JOIN `sys_dept` 获取 `dept_name`，或在前端维护一个 `deptId → deptName` 的映射字典。

**6. 文件预览增强**
当前佐证材料仅提供 `el-link` 下载。可集成图片全屏预览（`el-image` 的 `preview-src-list`），PDF 预览（`pdf.js` 或 iframe），Word 预览（`mammoth.js`），提升学术材料查看体验。

**7. 多文件上传优化**
当前 `fileUrl` 字段用逗号拼接多个 URL，缺少文件元信息。建议改为 JSON 格式存储：`[{name: "论文.pdf", url: "...", size: 1024, type: "pdf"}]`，前端展示时可显示文件名、大小、类型图标。

**8. 审核流程可视化**
为教师端增加"审核进度条"组件，用 `el-steps` 展示：提交 → 院级审核（通过/退回/驳回）→ 校级审核 → 最终通过。每个步骤显示审核时间、审核人、审核意见，方便教师追踪进度。

**9. 批量审核功能**
审核工作台增加表格多选 + 批量通过/驳回按钮。审核员日常可能面对数十条待审记录，逐条审核效率低下。后端需增加批量处理接口。

**10. 成果版本管理**
教师每次"退回修改→重新提交"后，旧版本内容被覆盖。建议增加版本快照表 `edu_achievement_version`，记录每次提交的完整内容，方便审核员对比修改前后差异。

### 🟢 技术优化（提升代码质量和系统稳定性）

**11. 单元测试覆盖**
核心 Service 层 `EduAchievementServiceImpl` 和 `EduAuditRecordServiceImpl` 没有任何测试用例。应至少覆盖：CRUD 操作、状态流转校验（如 status=0→1、1→2/4/5 等非法跳转检测）、聚合统计正确性。推荐使用 JUnit 5 + Mockito + H2 内存数据库。

**12. API 接口文档**
虽然 `edu-common-swagger` 模块已存在，但 `EduAchievementController` 和 `EduAuditRecordController` 未添加任何 `@Operation`、`@Tag`、`@Parameter` 注解。建议集成 Knife4j（基于 Springdoc 的增强 UI），为所有接口生成在线文档，便于前后端联调。

**13. 全局异常处理增强**
当前审核操作中的业务异常（如"成果不存在"、"状态不允许操作"）使用 `AjaxResult.error()` 返回，没有统一的业务异常码体系。建议定义 `BusinessException` + 错误码枚举（如 `ACHIEVEMENT_NOT_FOUND = 50001`），便于前端精确处理。

**14. 数据校验**
`EduAchievement` 和 `EduAuditRecord` 实体缺少 `@NotNull`、`@Size`、`@Pattern` 等 JSR 303 校验注解。Controller 方法也缺少 `@Validated` 注解。一旦前端绕过校验（如 Postman 直接调用），可能导致脏数据。

**15. 缓存策略**
统计接口 `/audit/statistics` 在每次请求时执行 4 条 SQL 聚合查询。数据量大时虽比内存循环快，但仍可进一步优化：用 Redis 缓存统计结果（TTL 5分钟），成果增删改时异步刷新缓存。

**16. 日志审计增强**
审核操作已使用 `@Log` 注解记录操作日志，但缺少审核流程的完整审计链：谁在什么时间对哪条成果做了什么操作、状态从 X 变为 Y。建议在 `edu_audit_record` 表中增加 `old_status` 和 `new_status` 字段。

**17. 安全加固**
- 教师的 CRUD 操作需校验 `teacherId`（当前 `teacherUpdateAchievement` 已有，但需确保一致性）
- 审核操作需校验审核员是否有权审核该学院的成果（院级审核员跨院审核的风险）
- `fileUrl` 字段存在 XSS 风险（URL 可能包含恶意脚本），需服务端过滤
- 登录"记住密码"使用 JSEncrypt 存 Cookie，建议评估安全性

**18. 前端代码质量**
- `achievement.js` 存在多余空行
- 多个 Vue 组件的 `<script>` 中使用压缩式写法（所有变量一行声明），降低可读性
- 缺少 ESLint 严格配置和 Prettier 格式化
- 组件缺少 `prop` 类型校验和默认值

**19. 性能监控**
集成 Spring Boot Admin（`edu-visual/edu-monitor` 已存在）但未配置告警规则。建议：
- 配置 JVM 内存告警阈值
- 配置 API 响应时间 P99 告警
- 添加健康检查端点 `/actuator/health` 的定期巡检

**20. 数据库索引优化**
`edu_achievement` 表的常用查询字段（`status`、`college_id`、`teacher_id`、`category`）可能缺少索引。建议用 `EXPLAIN` 分析慢查询并添加复合索引：
```sql
CREATE INDEX idx_achievement_status ON edu_achievement(status, del_flag);
CREATE INDEX idx_achievement_college ON edu_achievement(college_id, status);
CREATE INDEX idx_achievement_teacher ON edu_achievement(teacher_id, status);
```
