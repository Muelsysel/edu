# 系统权限对应指南 (RBAC Mapping Guide)

## 📌 更新记录 (Update Log)

  - **核心说明**：
    - 明确了 Navbar 通知组件必须依赖隐藏的 `system:notice:list` 及 `system:notice:edit` 权限才能拉取数据和执行已读操作；
    - 梳理了教师、院级和校级审核专家的操作流所需权限 (`achievement:achievement:teacherEdit`、`achievement:audit:college` 等)。

---

本指南详细列出了前端界面所属的功能模块、调用的核心API，以及后端对应的 `@RequiresPermissions` 权限控制字符。
在 RuoYi 的权限体系中，通常是在**菜单管理**中创建按钮类型的菜单项，并将对应的权限字符绑定在上面。然后在**角色管理**中为角色勾选这些菜单或按钮。

## 👨‍🏫 1. 教师端功能 (Teacher Role)

这是普通教师需要的功能，核心是成果的申报、查看与修改。

| 前端界面 / 功能点 | 前端 API 方法 | 后端接口 URL | 所需权限字符 `@RequiresPermissions` | 说明 |
| :--- | :--- | :--- | :--- | :--- |
| **我的成果管理** (首页/菜单) | - | - | `achievement:achievement:teacherList` (建议) | 控制左侧菜单栏是否显示 |
| **查询教师成果列表** | `teacherListAchievement` | `GET /achievement/achievement/teacherListAchievement` | `achievement:achievement:teacherQuery` | 教师获取自己提交的成果列表 |
| **查看成果详情** | `teacherGetAchievement` | `GET /achievement/achievement/teacherGetAchievement/{id}` | `achievement:achievement:teacherQuery` | 点击查看填报详情 |
| **新增成果填报** | `teacherAddAchievement` | `POST /achievement/achievement/teacherAddAchievement` | `achievement:achievement:teacherAdd` | 教师新增保存成果 |
| **修改成果** | `teacherUpdateAchievement` | `PUT /achievement/achievement/teacherUpdateAchievement` | `achievement:achievement:teacherEdit` | 教师在成果被退回或未提交时修改 |
| **删除成果** | `teacherDelAchievement` | `DELETE /achievement/achievement/teacherDelAchievement/{id}` | `achievement:achievement:teacherRemove` | 教师删除自己的成果 |
| **重新提交成果** | `teacherResubmit` | `PUT /achievement/achievement/teacherResubmit` | `achievement:achievement:teacherEdit` | 教师针对已驳回(状态4)的成果进行重提 |

## 🏢 2. 院级审核功能 (College Audit Role)

这是各学院秘书或负责人需要的功能。

| 前端界面 / 功能点 | 前端 API 方法 | 后端接口 URL | 所需权限字符 `@RequiresPermissions` | 说明 |
| :--- | :--- | :--- | :--- | :--- |
| **院级审核工作台** (菜单) | - | - | (看菜单表里的配置，例如 `achievement:audit:college`) | 控制工作台菜单显示 |
| **获取待审核列表** | `collegeAuditList` | `GET /achievement/audit/college/list` | `achievement:audit:college` | 获取状态为"待院级审核(1)"的成果 |

## 📊 4. 全局/通用查询扩展查询 (General & Admin)

这些功能通常分配给管理员，或开放给校级人员和部分管理人员。

| 前端界面 / 功能点 | 前端 API 方法 | 后端接口 URL | 所需权限字符 `@RequiresPermissions` | 说明 |
| :--- | :--- | :--- | :--- | :--- |
| **全部成果列表管理** | `listAchievement` | `GET /achievement/achievement/list` | `achievement:achievement:list` | 管理员查看全校所有人提交的成果 |
| **新增/编辑/删除** (全局) | `addAchievement` 等 | `POST/PUT /achievement/achievement` | `achievement:achievement:add/edit/remove` | 管理员上帝视角的强行增删改 |
| **查询审核记录** | `listAuditRecord` | `GET /achievement/audit/record/list` | `achievement:audit:recordList` | 独立页面查看所有审核的历史轨迹 |
| **获取成果审核进度** | `getAuditProgress` | 同上 | `achievement:audit:recordList` | 点击 el-steps 进度条时请求数据 |
| **导出审核记录** | - | `POST /achievement/audit/record/export` | `achievement:audit:export` | 导出 Excel |
| **首页数据统计面板** | `getStatistics` | `GET /achievement/audit/statistics` | 无权限限制（或依赖于外层页面权限拦截）| Dashboard 调用的接口，目前后端并未使用注解拦截 |

## 🔔 5. 系统级依赖权限 (System Dependencies)

系统基础服务的接口调用权限，**只要登录且进入系统首页，通常都需要这些权限**：

| 模块 | 取数场景 | 前端 API 方法 | 后端接口 URL | 所需权限字符 `@RequiresPermissions` | 解决问题 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **消息通知** | Navbar 获取右上角角标与下拉消息列表 | - | `GET /system/notice/list` | `system:notice:list` | 不给这个权限，角标出不来且报错403 |
| **通知确认操作** | Navbar 点击“我知道了” | `updateNotice` | `PUT /system/notice` | `system:notice:edit` | （由于原后端可能设计，修改消息同需edit权限，如果没有可能会操作失败） |
| **字典获取** | 前端各种下拉选择框（如所属学院、成果类型） | `getDicts` | `GET /system/dict/data/type/{dictType}` | `无强制拦截` 或 `system:dict:list` | (大部分情况下 RuoYi 默认放开基本字典查询，或使用 `@Anonymous`) |

## 💡 配置建议
1. 教师无需分配任何 `audit` 相关的权限，避免不安全的越权。
2. 对于 **消息通知 (`system:notice:list` 和 `system:notice:edit`)**，如之前所述，可以在“菜单管理”中将属于这俩权限的按钮（或父级菜单“通知公告”）的**显示状态设置为空/隐藏**，然后再在“角色管理”中为教师/审核员打勾。这样他们既拥有了调用接口的凭证，又不会在左侧看到冗余的主菜单项。
